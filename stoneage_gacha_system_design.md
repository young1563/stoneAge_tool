# StoneAge Idle Gacha System Analysis Tool

## 게임 시스템 기획 포트폴리오 설계 문서

------------------------------------------------------------------------

# 1. 프로젝트 개요

## 1.1 프로젝트 목적

본 프로젝트는 모바일 게임 **"스톤에이지 키우기"의 확률 데이터를 기반으로
가챠 시스템 구조를 분석하고 시뮬레이션하는 데이터 기반 게임 시스템 분석
도구**를 제작하는 것을 목표로 한다.

단순한 데이터 시각화를 넘어 다음을 수행한다.

-   확률 테이블 구조 분석
-   가챠 기대값 계산
-   유저 체감 확률 분석
-   Monte Carlo 기반 시뮬레이션
-   매출 기대값 추정
-   시스템 설계 개선 제안

이를 통해 **게임 시스템 기획자의 데이터 기반 분석 역량**을 포트폴리오로
보여주는 것을 목표로 한다.

------------------------------------------------------------------------

# 5. 시스템 분석 항목

------------------------------------------------------------------------

# 5.1 확률 구조 분석

## 목적

가챠 시스템의 전체 확률 구조를 파악한다.

## 분석 항목

-   등급별 등장 확률
-   확률 분포
-   희귀도 구조
-   확률 밸런스

## 예시

SS 등급 확률

0.0075%

------------------------------------------------------------------------

# 5.2 레벨 기반 확률 변화 분석

## 목적

플레이 진행도에 따른 확률 변화를 분석한다.

## 분석 내용

-   레벨별 확률 변화
-   확률 성장 곡선
-   progression 설계 의도

### 그래프 구조

X = 레벨\
Y = SS 등장 확률

------------------------------------------------------------------------

# 5.3 픽업 가챠 분석

## 목적

픽업 시스템이 확률에 미치는 영향을 분석한다.

## 분석 항목

-   일반 SS 확률
-   픽업 SS 확률
-   확률 증가 배율

### 예시

일반 SS 확률 = 0.0075%\
픽업 SS 확률 = 0.12%

약 16배 증가

------------------------------------------------------------------------

# 5.4 유저 체감 확률 분석

## 목적

유저 입장에서 체감되는 확률을 분석한다.

## 확률 계산

P = 1 - (1 - p)\^n

p = 단일 가챠 확률\
n = 뽑기 횟수

### 예시

  뽑기 횟수   SS 획득 확률
  ----------- --------------
  100         7%
  300         20%
  1000        53%

------------------------------------------------------------------------

# 5.5 Monte Carlo 시뮬레이션

## 목적

확률 시스템을 실제 가챠 상황으로 시뮬레이션한다.

## 방법

랜덤 샘플링 기반 확률 시뮬레이션

## 시뮬레이션 항목

-   10연차
-   100연차
-   1000연차

## 결과

-   평균 획득 횟수
-   확률 분포
-   기대 확률

------------------------------------------------------------------------

# 5.6 매출 기대값 분석

## 목적

가챠 시스템의 매출 구조를 분석한다.

## 계산식

Expected Draw = 1 / p

Expected Cost = Expected Draw × Draw Price

### 예시

SS 확률 = 0.0075%\
평균 뽑기 = 13,333

1회 뽑기 가격 = 300원

예상 비용 ≈ 4,000,000원

------------------------------------------------------------------------

# 6. 시스템 문제점 분석

## 6.1 체감 확률 문제

SS 확률이 매우 낮아 유저 체감 확률이 낮을 가능성이 있음.

## 6.2 확률 이해도 문제

확률 구조가 복잡할 경우 유저가 시스템을 이해하기 어려움.

------------------------------------------------------------------------
